Talks that shaped my programming

  • Mike Acton: Data Oriented Programming
    • Programmers job is not write code;
      Programmers job is to solve (data transformations) problems
    • Understand the data to understand the problem
  • Brian Will: Why OOP is Bad
  • What is Systems Programming, Really?
  • Eskil Steenberg: How I program C
    • In the beginning you always want Results.
      In the end all you want is control.
  • Hammock Driven Development - Rich Hickey
  • Jeff and Casey's Guide to Becoming a Bigger Programmer
    • Spend time in the debugger with disassembled code
    • Write the code with "some room to breathe". Write the code you will use.
    • Be more systematic when fixing bugs. e.g Keep track of the type of bugs
    • Being a better programmer means being good at tradeoffs, no code will be good enough.
  • The Silver Bullet Syndrome by Hadi Hariri
    • There is no silver bullets, no languages or frameworks. Use the ones which you are comfortable and have better tooling.
